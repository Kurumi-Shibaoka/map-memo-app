Database Schema06

MVCパターンとDAO/DTO構成

MVCパターン

  • Controller Servletで制御
  • View JSP + JSTLで画面表示
  • Model DAO/DTOでデータ管理
  • DB接続は DBConnection で共通化

Restaurant DAO / DTO

  • DTO 店舗情報の受け渡す Restaurant クラス
  • DAO DBのCRUD処理をまとめた DAO クラスと DBConnection クラスに分離
try-with-resources により例外発生時も自動でリソースを安全に閉じる

画面遷移図

Screen Transition Diagram08

List 画面：一覧・検索機能の実装

機能解説

  • ListServlet でDBからデータを取得
  • 検索時は findByCondition で絞り込み
  • JSTL <c:forEach> で店舗を表示
  • <c:if> で「お気に入り(💖)」等を表示
List View Implementation09

Add 画面：新規登録のフロー

実装のポイント

  • Encoding: UTF-8設定による文字化け防止
  • Registration: doPostでDTOに格納して登録
  • Handling: チェックボックス未選択時のnull対策
  • Redirect: 登録後はリダイレクト 二重送信防止
Add View Implementation10

Edit 画面：編集・削除機能

ステータス管理

  • Data: findByIdForEdit で編集対象を取得
  • JSTL: フォームにデータを表示
  • Safety: 削除前に確認ダイアログ
  • Redirect: 更新・削除後は一覧へ戻る
Edit & Delete Flow11

Map 画面：Googleマップによる位置表示

地図表示の仕組み

  • Parameter: URLパラメータでID取得
  • DAO: 店舗情報をDBから取得
  • Display: mapUrlをiframeで表示

地図連携のメリット

  • Googleマップの埋め込みURLを利用
  • 地図表示機能を簡単に実装
